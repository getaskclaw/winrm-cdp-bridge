# Host facts template

Copy this file, rename to `host.env`, fill in your values. Never commit the real file.

```env
host=<tailscale-ip-or-hostname>
user=<windows-username>
pass=<windows-password>
```

## Credential file format

The scripts read credentials from a file specified by `--credentials` or `$WINRM_CREDENTIALS`. Default lookup paths are not hardcoded — you must pass the path explicitly.

## Connection reference

| Protocol | Default port | Notes |
|---|---|---|
| WinRM HTTP | 5985 | Default. Use NTLM transport. No encryption without HTTPS. |
| WinRM HTTPS | 5986 | Recommended — pinned cert or trusted CA required. |
| RDP | 3389 | Interactive desktop access. |
| SMB | 445 | File transfer over network share. |
| SSH (OpenSSH) | 22 | Alternative if WinRM is unreliable. |

**Security:** Do not expose WinRM (5985/5986) or RDP (3389) beyond your Tailnet ACL. Prefer WinRM HTTPS over HTTP. Use a dedicated non-admin service account, not a local administrator with `LocalAccountTokenFilterPolicy=1`. Rotate passwords regularly.
